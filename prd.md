## 项目概述

 **产品** ：Verbatim（临时名） — 本地 PDF 对比工具

 **目标用户** ：制药、法规、质量管理等需高精度文本比对的行业用户

 **核心价值** ：在保持原始视觉的前提下，提供字符级 + 格式级差异比对，支持人工参与的区域对齐，输出直观可审计的差异报告。

## 主要功能列表（MVP）

1. 文件加载
   * 加载左/右两份 PDF（支持多页）
2. 页面渲染
   * 高分辨率渲染页面为图片（用于 UI 标注）
3. 区域选择与映射
   * 用户在左右页面上手动框选区域并建立映射（支持 1:N）
   * 保存/加载映射（JSON）
4. 字符级文本差异
   * 支持空格敏感、换行不敏感的字符级 diff
   * 返回每个差异的字符索引与 bbox
5. 格式差异检测
   * 字体、字号、颜色、bold/italic 的差异检测（采用冻结阈值）
6. 视觉高亮与连接线
   * 文本差异：单字符红色半透明背景 + 区域红色实线框 + 角标（T: N）
   * 格式差异：单字符绿色虚线框 + 区域绿色虚线框 + 角标（F: N）
   * 同字符既有文本又有格式差异：红底 + 绿色虚线框叠加
7. 报告导出
   * 导出 JSON 与 PDF 报告（带差异截图与映射信息）
8. 本地日志与可复现映射
   * 所有映射与 diff 操作保存，便于审计与回溯

## 用户故事与验收标准

### 用户故事 1：加载并浏览 PDF

* 场景：用户打开工具并加载左侧 17 页原版 PDF 与右侧 2 页缩印版
* 验收：页面渲染速度合理（首屏渲染 < 2s，单页缩略图生成 < 1s）

### 用户故事 2：建立区域映射

* 场景：用户在左侧第 3 页选区并在右侧第 1 页选区建立对应
* 验收：`<span>mappings/*.json</span>` 记录映射，后续可加载并复现

### 用户故事 3：查看差异

* 场景：用户对一个映射点击“比对”按钮
* 验收：界面高亮显示文本差异（红）、格式差异（绿），角标显示数量，且差异可定位到字符

### 用户故事 4：导出报告

* 场景：用户导出当前映射的差异报告
* 验收：生成 JSON（包含 ops 列表）与 PDF（带截图与标注）

## UI/交互设计要点

* 左右并列 viewer（同步缩放/平移可选）
* 顶部工具栏：加载文件、保存映射、加载映射、导出报告、打包/关于
* 区域列表侧边栏：列出已建立的映射（支持重命名/删除/跳转）
* 差异详情面板：显示选中映射的 ops 列表（可点击跳转到对应字符）

## 性能要求

* 单次 region diff（平均 2000 字符）目标 < 300ms
* 渲染高分截图（A4 300dpi）不超过 1s（取决于硬件）

## 风险与缓解

* 风险：PDF 内部字体信息不完整 → 缓解：回退到字符渲染比对（渲染后 OCR 仅作补充，但第一版仍禁止公开 OCR）
* 风险：复杂版式导致大量人工配对 → 缓解：提供复制/拆分选区工具，批量建立映射辅助
