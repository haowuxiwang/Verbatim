# 缩放功能 PRD（草案）

## 1. 背景与目标
用户导入文档后，常因字体小或版面密集导致框选困难。  
在“开始对比前”提供缩放能力，提升选区精度和操作效率。

## 2. 范围
- 左右文档预览区支持缩放。
- 导入后即可缩放，对比前后都可使用。
- 左右视图缩放独立。

## 3. 功能需求
1. 左右视图独立支持: 放大、缩小、重置 100%。
2. 默认初始比例: 100%（保持当前导入后默认视觉一致）。
3. 显示当前缩放比例。
4. 支持快捷方式: `Ctrl + 滚轮`（当前焦点视图）。
5. 缩放不改变业务比对逻辑，仅影响显示和坐标映射。

## 4. 非功能需求
1. 可用性: 不缩放时行为与现版本一致。
2. 性能: 连续缩放时不可出现明显卡死（>500ms 主线程冻结）。
3. 精度: 不同比例下同一区域映射到 PDF 坐标保持一致。
4. 稳定性: 缩放后翻页/加载历史选区/OCR路径均可用。

## 5. 交互规范
- 比例范围: 25% ~ 400%
- 步进: 10%
- 控件: `[-] [xx%] [+] [100%]`（左右各一组）
- 到达上/下限时对应按钮置灰

## 6. 验收标准
1. 导入后左右默认均为 100%。
2. 左右缩放互不影响。
3. 任意缩放比例下框选文本提取一致（同一实际区域）。
4. 缩放不会影响 Compare 按钮可用状态判断。
5. 对比结果不因仅视图缩放而改变。

## 7. 风险
1. 缩放与坐标映射不一致导致框选偏移。
2. 高频缩放触发重渲染导致卡顿。
3. 差异高亮层若仍使用单一缩放参数可能错位。

## 8. 里程碑
1. M1: 状态与控件（独立缩放、比例显示、快捷键）
2. M2: 坐标映射与高亮修正
3. M3: 回归测试与文档更新
