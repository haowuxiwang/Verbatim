# P0 问题清单（执行后）

日期: 2026-03-04

## 高优先级

1. OCR 侧坐标高亮能力有限（策略已确认）
- 现状: OCR 回退后可稳定得到文本，但 OCR 侧精确坐标映射不稳定，标注不完整。
- 影响: 差异“可读性”仍可接受，但“可定位性”弱于文本层侧。
- 建议: 保持当前“文本级标注”为默认；在高级设置保留实验开关，不作为发布阻塞项。

2. 文本质量判定存在个别误报风险
- 现状: `original.pdf` 第 14 页被判为 `quality=bad`（乱码信号强）。
- 风险: 可能出现“文本层可用但被建议 OCR”的误报，影响性能和结果一致性。
- 建议: 引入“页内区域上下文”二次校验，避免仅靠文档级统计直接强制 OCR。

## 中优先级

1. 控制台/源码存在部分中文乱码迹象（可维护性风险）
- 现状: 个别源文件字符串在终端查看时出现 mojibake。
- 影响: 维护与排障成本上升，日志可读性受影响。
- 建议: 统一 UTF-8 编码、增加编码校验 pre-commit 规则。

2. GUI 端到端自动化覆盖仍不足
- 现状: 逻辑层测试较完整，GUI 交互链路仍以人工验证为主。
- 影响: UI 重构时回归风险较高。
- 建议: 增加最小 GUI smoke（加载、框选、触发比对、差异列表出现）自动化用例。

## 低优先级

1. 性能 SLA 仍未工程化固化
- 现状: 有性能测试，但未形成“文件规模 -> 时延上限”的发布门禁。
- 建议: 在 CI 增加基准阈值与趋势监控。
