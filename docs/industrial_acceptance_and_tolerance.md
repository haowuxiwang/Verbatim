# 工业化验收标准 + 容错策略（执行版）

更新时间：2026-03-04  
适用范围：`Verbatim_dev` 本地开发版到可灰度 `.exe` 分发前

## 1. 验收目标（必须量化）

### A. 可靠性
- 同一选区连续比对 3 次：`Diff ops` 波动不超过 `±1`
- 已知一致样例：误报率 `<= 5%`
- 已知差异样例：漏报率 `<= 10%`

### B. 稳定性
- 单次比对 P95 耗时 `<= 10s`
- OCR 超时场景不崩溃，不出现后台线程析构错误
- 低可信 OCR 不输出硬结论，统一降级 `REVIEW`

### C. 可解释性
- 每次结果必须包含：`状态(PASS/REVIEW)`、`可信度(高/中/低)`、`OCR是否参与`
- `REVIEW` 必须带明确原因（超时/低置信/高相似噪声）

### D. 可追溯
- 每次比对结果记录到历史（至少 200 条）
- 记录字段：时间、状态、差异数量、OCR参与、页码、选区、告警数、摘要

## 2. 容错策略（后端决策）

### A. OCR 路由
- 默认：`Paddle云端优先`
- 云端失败时：根据配置决定是否启用本地 OCR 兜底
- 本地 OCR 采纳前执行稳定性双跑校验

### B. 结果门控
- OCR 参与且低可信时：`REVIEW`，不发布硬差异
- 高相似文本（OCR场景）出现微差异：优先按噪声处理
- OCR/低质量场景默认关闭字段映射，避免放大误报

### C. 用户非理想操作容错
- 小选区、错位、跨栏时，优先触发复核提示而非强结论
- 明确引导“放大后重选/扩大选区/切换页面模式”
