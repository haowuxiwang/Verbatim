# GUI 体验优化 TodoList（执行版）

更新时间: 2026-03-04

## P0 可用性与可见性

- [x] 恢复按钮可点击边界
  - 目标: 让用户一眼识别可操作控件
  - 验收: 顶栏 `QPushButton/QToolButton` 有清晰边框、hover 反馈

- [x] 左右文档空态改为独立上传卡片
  - 目标: 用主画布中部入口替代“请点击左/右侧PDF”纯文本提示
  - 验收: 左侧显示“上传左侧PDF”，右侧显示“上传右侧PDF”，且可直接触发对应上传流程

- [x] 上传卡片交互态
  - 目标: 降低用户迷茫，强化操作层级
  - 验收: 默认浅灰背景+边框，hover 边框变蓝，按下背景加深

## P1 布局与流程

- [x] 顶栏保持单行，避免双行拥挤
  - 目标: 提升扫描效率与首屏清晰度
  - 验收: 文档控制/系统设置/开始对比在同一行

- [x] 顶栏移除左右PDF入口按钮
  - 目标: 避免与主画布上传卡片形成双主入口冲突
  - 验收: 顶栏不再出现 `左侧PDF/右侧PDF` 按钮，改由左右独立上传卡片承担上传入口

- [x] 次要比对开关放入高级设置
  - 目标: 减少高频界面噪声
  - 验收: `显示格式差异`、`忽略空白` 不在主栏展示，可在 `高级设置` 中调整

- [x] 上传文案泛化为“文档”
  - 目标: 为后续多格式比对扩展预留语义空间
  - 验收: 卡片按钮为 `上传左侧文档` / `上传右侧文档`

- [x] 左右翻页/缩放控制下沉至左右独立文档卡片
  - 目标: 减少顶栏拥挤，符合“每侧独立控制、就近操作”的交互模型
  - 验收: 顶栏不再承载左右分页/缩放；每侧文档卡片头部有本侧分页与缩放控件

## P2 稳定性

- [x] 修复 splitter 控件越界警告
  - 目标: 消除启动日志 `QSplitter::setCollapsible: Index out of range`

- [x] Compare 按钮状态日志去重
