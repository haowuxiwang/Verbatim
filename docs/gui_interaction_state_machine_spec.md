# GUI交互状态机与语义规范（P1）

更新时间: 2026-03-04  
范围: `app/main_window.py` 主界面交互、缩放/框选/比对/差异标注

## 1. 目标

1. 让用户始终知道“当前处于哪个操作模式、结果是否可信、下一步该做什么”。
2. 消除缩放/翻页/切换模式后标注语义漂移（尤其是“差异焦点”被误解为“整块选区”）。
3. 把 GUI 交互约束成可测试的有限状态机，直接支撑自动化测试。

## 2. 术语与显示语义

1. 选区层（Selection Layer）
- 含义: 用户框选的候选比对范围。
- 视觉: 半透明蓝/绿框。
- 生命周期: 导入、翻页、重选区会更新；比对后可隐藏但不应冒充差异。

2. 差异层（Diff Layer）
- 含义: 比对算法输出的差异区域（内容/格式）。
- 视觉: 内容差异与格式差异颜色分离，支持过滤。
- 生命周期: 仅由“开始对比”生成；缩放/重绘必须保持几何一致。

3. 焦点层（Focus Layer）
- 含义: 用户点击某条差异后的当前关注对象。
- 视觉: 高饱和边框 + 自动滚动到视口。
- 生命周期: 点击差异后生效；缩放后必须重算位置并保持焦点。

## 3. 核心状态机

状态定义:

1. `S0_IDLE`
- 条件: 左右文档未齐备。
- Compare: 禁用。

2. `S1_DOCS_READY`
- 条件: 左右文档已加载，未完成双侧选区。
- Compare: 禁用。

3. `S2_SELECTION_READY`
- 条件: 左右页 + 左右选区齐备。
- Compare: 启用。

4. `S3_COMPARING`
- 条件: 正在执行比对/OCR回退。
- Compare: 禁用，显示等待状态。

5. `S4_RESULT_READY`
- 条件: 差异结果已生成并渲染。
- Compare: 启用（允许再次比对）。
- 支持: 差异点击、焦点联动、缩放重绘。

6. `S5_RESULT_STALE`
- 条件: 翻页/换文档/重选关键输入后，历史结果失效。
- Compare: 依据选区就绪与否。
- 行为: 清空缓存差异与焦点。

## 4. 事件-动作-迁移

1. `E_LOAD_LEFT` / `E_LOAD_RIGHT`
- 动作: 更新页码、清理失效结果缓存。
- 迁移: `S0 -> S1` 或保持 `S1`。

2. `E_SELECT_LEFT` / `E_SELECT_RIGHT`
- 动作: 更新选区 bbox，刷新 compare 按钮。
- 迁移: 双侧齐备时 `S1 -> S2`。

3. `E_COMPARE`
- 前置: `S2`。
- 动作: `S3`，完成后写入差异缓存（ops/regions/ocr_flags），渲染差异层。
- 迁移: `S3 -> S4`。

4. `E_CLICK_DIFF_ITEM`
- 前置: `S4`。
- 动作: 设置焦点层，滚动到对应位置。
- 迁移: 保持 `S4`。

5. `E_ZOOM_LEFT` / `E_ZOOM_RIGHT`
- 前置: `S4` 时必须“重算差异层 + 保持焦点层”；否则仅缩放选区层。
- 迁移: 保持原状态。

6. `E_PAGE_CHANGE` / `E_DOC_REPLACE`
- 动作: 清空差异层、焦点层、缓存；选区按规则清理。
- 迁移: `S4 -> S5 -> S1/S2`。

## 5. 不变量（必须满足）

1. 缩放不改变差异语义: 同一 diff op 仅做坐标重算，不得退化成整块选区高亮。
2. 翻页后不得复用旧差异: 任一页号变化后缓存 diff 必须失效。
3. 焦点优先级高于选区: 有焦点时视觉上突出焦点，而不是选区。
4. OCR标注语义可解释: OCR无稳定映射时仅显示文本级差异，不显示误导性坐标块。

## 6. 错误与降级策略

1. OCR结果可用但坐标不可信
- 策略: 差异列表可展示，文档上仅显示“低置信提示”，不做伪精确框。

2. 选区文本过少/为空
- 策略: 禁止进入比对或给明确提示，不允许“成功但无语义结果”的状态。

3. 缩放重绘失败
- 策略: 保留上一次差异层并提示重试，不自动回退为整块选区。

## 7. 验收标准（P1）

1. 缩放 5 次后，焦点差异仍定位在同一词/短语，不扩张为整块选区。
2. 翻页后点击旧差异项不会产生高亮（旧结果已失效）。
3. Compare 按钮状态与输入前置一致，不出现“可点但无选区”路径。
4. 小屏（<=1366 宽）控件不重叠，主操作路径可见可点。
