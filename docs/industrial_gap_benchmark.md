# 工业化差距对标表

更新时间: 2026-03-04  
范围: Verbatim 本地开发版到可分发工业级 `.exe`

| 领域 | 现状 | 目标（工业级） | 优先级 | 预计工作量 |
|---|---|---|---|---|
| 发布与升级 | 可打包 `.exe`，但缺完整发布流程与回滚演练 | 建立发布清单、灰度策略、回滚/迁移文档 | P0 | 3-5 天 |
| GUI 端到端回归 | 以单元测试为主，GUI 自动化覆盖不足 | 建立 GUI 回归矩阵并纳入 CI | P0 | 5-8 天 |
| 性能与容量SLA | 有性能测试但无正式 SLA | 定义 SLA（文件大小/页数/时延）并固化压测 | P0 | 3-5 天 |
| 可观测性与故障处理 | 已有结构化日志基础与错误码草案 | 完成错误码分级、排障手册、支持流程 | P1 | 3-4 天 |
| 安全与合规 | Token 已加密，日志脱敏初步完成 | 形成数据分级、留存策略、审计规范 | P1 | 4-6 天 |
| 依赖治理 | 已拆 runtime/build/dev，仍需锁文件收敛 | 最小依赖锁定、漏洞扫描、SBOM | P1 | 3-5 天 |
| 架构可维护性 | 入口已拆分，主窗口模块仍偏大 | 继续分层拆解与模块化测试 | P1 | 5-10 天 |
| 包体与分发体验 | 可构建，但包体/冷启动未系统优化 | 优化包体、启动耗时和安装体验 | P2 | 3-6 天 |
| 文档体系 | 有开发文档，用户与运维文档不足 | 用户手册、FAQ、支持SOP 完整 | P2 | 2-4 天 |
| 质量门禁 | 测试通过，但门禁策略未完全固化 | CI 门禁（覆盖率/性能/构建验收）标准化 | P1 | 2-4 天 |
