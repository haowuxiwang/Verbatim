# Review Archive

This directory stores historical review material, stage summaries, fail logs, and execution todo notes that are useful for forensic tracing but are not active source inputs.

## What belongs here

- dated `sum`, `review`, `faillog`, and `todolist` markdown files
- one-off stage findings and historical execution notes
- retrospective records that explain why the repo evolved a certain way

## What does not belong here

- active source code
- current build entrypoints
- runtime state
- release artifacts
