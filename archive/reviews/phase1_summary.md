# 第一阶段总结（Verbatim_dev）

## 已完成事项
1. 差异噪音治理（泛化规则，非单用例硬编码）
- 增加字段化启用门控：仅在结构化证据充分时启用字段比对；叙述段落自动跳过字段化，减少误报。
- 增加字段值鲁棒判等：URL 归一、电话/传真粘连容忍、常见符号丢失容忍。
- 增加低置信文本层下的微小差异压制，降低脏文本层碎噪声。

2. 阅读顺序与布局优化
- `auto` 阅读顺序从单一规则升级为综合评分（质量+差异量）选择。
- 双栏检测阈值下调，支持中等选区触发双栏判定（更贴近手工框选场景）。

3. GUI 乱码与交互优化
- 修复差异列表、详情、选区历史等关键可见乱码问题。
- 主界面简化：将复杂参数收纳到“高级设置”。
- 翻页交互改进：`上一页/下一页` 明确文案，页码显示改为“共 N 页”。

4. OCR 接入（主路径已打通）
- 接入 Paddle OCR：同步主路径 + 异步接口预留。
- token 未硬编码：支持环境变量与 GUI 设置文件。
- 支持文档级质量评估并可单侧强制 OCR（混合文档：一页扫描一页正常也可处理）。
- 扫描页不再直接报错覆盖页面，保留框选与比对流程。

5. 工程验证
- 当前测试集通过（最近一轮：`32 passed`）。
- `py_compile` 通过。
- 本阶段按要求未进行最终打包发布流程。

---

## 未完成事项
1. OCR 网络链路稳定性（高优先级）
- 代理/证书环境下存在 SSL 握手失败（`UNEXPECTED_EOF_WHILE_READING`），需增强网络层诊断与兼容策略（代理显式配置、重试退避、TLS/证书策略）。

2. OCR 成功后的定位高亮
- 当前 OCR 回退文本可用于内容对比，但缺少精确字符级 bbox 映射，导致高亮定位能力受限。

3. 选区历史治理
- 增加“清空历史/迁移旧历史编码”入口，避免旧记录影响新用户体验。

4. 体验增强
- 可在 GUI 顶部展示“文档级策略状态”（如“右侧：强制OCR”），减少用户依赖日志判断。

---

## 已知问题
1. OCR 在当前代理网络下可能触发但失败
- 现象：日志出现 `OCR decision`，但随后 `OCR failed ... SSL EOF`，最终仍回落到文本层对比。
- 影响：噪音观感接近未启用 OCR。

2. 历史记录可见“生产企业”等旧内容并非硬编码
- 来源：`mappings/region_selections.json` 的本地旧数据。
- 影响：用户可能误判为系统内置模板或硬编码。

3. 脏文本层场景仍有残余噪音
- 在 OCR 未成功时，缩印双栏与混排文本层错序/错字会继续带来噪音。

4. OCR 回退模式下格式高亮能力受限
- 无完整文本层坐标时，格式差异定位不如原生文本层精确。

---

## 第一阶段结论
本阶段已完成“可用性底座”建设：人机协同框选、泛化降噪、GUI 可用性与 OCR 接入策略。  
当前阻塞点集中在“真实网络环境下 OCR 请求稳定性”，解决后将显著提升扫描件与低质量缩印件场景的可用性和一致性。
